```Flow Execute query `limit` Save `nextCursor value result Pass `nextCursor `after parameter next page```typescript

  limit: 20 
})

  limit: 20,
  after: items.nextCursor 
})
```Infinite Scroll Subscriptions query ZyncBase manages live window load older data without breaking sync```typescript
const sub = client.store.subscribe('items', {
  orderBy: { created_at: 'desc' },
  limit: 20
}, (items) => {
  render(items)
})

// Load 20 more historical items into the 'items' array
if (sub.hasMore) {
  await sub.loadMore()
}

```Operator Reference Table Operator Type Description Example `eq` Equal to `{ status eq 'active' `ne` Not equal to `{ status 'deleted' `gt` Number/Date Greater than `{ age: gt: 18 `gte` Number Greater or equal `{ age gte 18 `lt` Number/Date than `{ age: lt: 65 } `lte` Number Less than or equal `{ age: lte: 65 } `contains` substring `{ name contains 'john' } `startsWith` Starts with prefix `{ name startsWith 'J' `endsWith` Ends with suffix `{ name endsWith 'son' `in` Value in array `{ role'admin 'editor' `notIn` Value not in array `{ role notIn'guest' `isNull` null/undefined `{ deleted_at isNull: true } `isNotNull` value `{ verified_at isNotNull: true } Best Practices Use implicit simplicity```typescript
// Good - clean and readable
{
  where: {
    age: { gte: 18 },
    status: { eq: 'active' }
  }
}

// Avoid - unnecessary complexity
{
  where: {
    and: [
      { age: { gte: 18 } },
      { status: { eq: 'active' } }
    ]
  }
}
```Prefer `in` over conditions```typescript
// Good - concise
{
  where: {
    status: { in: ['active', 'pending', 'review'] }
  }
}

// Avoid - verbose
{
  where: {
    or: [
      { status: { eq: 'active' } },
      { status: { eq: 'pending' } },
      { status: { eq: 'review' } }
    ]
  }
}
```paginate large result sets```typescript
// Good - prevents memory issues
{
  where: { status: { eq: 'active' } },
  limit: 100
}

// Risky - could return thousands of items
{
  where: { status: { eq: 'active' } }
}
```Use specific queries broad```typescript
// Good - targeted query
{
  where: {
    project_id: { eq: currentProject },
    status: { eq: 'active' }
  }
}

// Avoid - fetches too much data
{
  where: {
    status: { eq: 'active' }
  }
}
